实例 1
解决方案
不要相信客户端输入，即使有客户端验证。

通常，在服务器端对所有的数据进行类型检查。

如果应用程序使用 JDBC，则使用 PreparedStatement 或 CallableStatement，参数由“?”传递

不要使用简单的字符串连接创建动态 SQL 查询。

转义从客户端收到的所有数据。

在用户输入中使用 允许字符的“允许列表”或不允许字符的“拒绝列表”。

通过尽可能使用最低权限的数据库用户来应用最低权限原则。

尤其要避免使用“sa”或“db-owner”数据库用户。 这并不能消除 SQL 注入，但可以将其影响降至最低。

授予应用程序所需的最少数据库访问权限。
